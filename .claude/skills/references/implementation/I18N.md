# Internationalization (i18n) Patterns

Patterns for building multilingual, globally-ready applications.

## Overview

Internationalization enables your app to support multiple languages and regions. Key concerns:
- **Translation** — Text in user's language
- **Formatting** — Numbers, dates, currencies
- **Layout** — RTL support, text expansion
- **Cultural** — Icons, colors, imagery

---

## Translation Architecture

### Message Format

```tsx
// ✗ Bad: Concatenation (breaks in other languages)
const message = 'Hello, ' + name + '! You have ' + count + ' messages.';

// ✓ Good: ICU message format with placeholders
const message = t('greeting', { name, count });
// en: "Hello, {name}! You have {count} messages."
// de: "Hallo, {name}! Sie haben {count} Nachrichten."
```

### File Structure

```
locales/
├── en/
│   ├── common.json      # Shared translations
│   ├── auth.json        # Authentication
│   ├── dashboard.json   # Dashboard-specific
│   └── errors.json      # Error messages
├── de/
│   ├── common.json
│   ├── auth.json
│   └── ...
└── ja/
    └── ...
```

### Translation Keys

```json
// ✗ Bad: Generic keys
{
  "button1": "Submit",
  "text1": "Welcome back"
}

// ✓ Good: Descriptive, namespaced keys
{
  "auth.login.submit": "Sign in",
  "auth.login.welcomeBack": "Welcome back, {name}",
  "dashboard.stats.totalUsers": "Total users"
}
```

## Pluralization

### ICU Plural Rules

```json
{
  "items.count": "{count, plural, =0 {No items} one {# item} other {# items}}",
  "days.remaining": "{days, plural, =0 {Today} one {# day left} other {# days left}}"
}
```

### Complex Pluralization (Select)

```json
{
  "notification.new": "{count, plural, =0 {No new notifications} one {{gender, select, male {He} female {She} other {They}} sent you a message} other {# new notifications}}"
}
```

### Language-Specific Rules

```
English: one, other (1 item, 2 items)
Russian: one, few, many, other (1 товар, 2 товара, 5 товаров)
Arabic: zero, one, two, few, many, other
Japanese: other (no plural forms)
```

## Number Formatting

### Basic Number Formatting

```tsx
// Use Intl.NumberFormat, not hardcoded formatting
const formatter = new Intl.NumberFormat(locale);

// Examples by locale:
// en-US: 1,234,567.89
// de-DE: 1.234.567,89
// fr-FR: 1 234 567,89
// ja-JP: 1,234,567.89
```

### Currency Formatting

```tsx
function formatCurrency(amount: number, currency: string, locale: string) {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(amount);
}

// formatCurrency(1234.56, 'USD', 'en-US') → "$1,234.56"
// formatCurrency(1234.56, 'EUR', 'de-DE') → "1.234,56 €"
// formatCurrency(1234.56, 'JPY', 'ja-JP') → "￥1,235"
```

### Percentage and Unit Formatting

```tsx
// Percentage
new Intl.NumberFormat(locale, {
  style: 'percent',
  minimumFractionDigits: 1,
}).format(0.1234);
// en-US: "12.3%"
// de-DE: "12,3 %"

// Units
new Intl.NumberFormat(locale, {
  style: 'unit',
  unit: 'kilometer',
  unitDisplay: 'short',
}).format(100);
// en-US: "100 km"
// de-DE: "100 km"
```

## Date and Time Formatting

### Date Formatting

```tsx
function formatDate(date: Date, locale: string, options?: Intl.DateTimeFormatOptions) {
  return new Intl.DateTimeFormat(locale, options).format(date);
}

// Short date
formatDate(date, 'en-US', { dateStyle: 'short' });  // "1/15/24"
formatDate(date, 'de-DE', { dateStyle: 'short' });  // "15.01.24"
formatDate(date, 'ja-JP', { dateStyle: 'short' });  // "2024/01/15"

// Full date
formatDate(date, 'en-US', { dateStyle: 'full' });
// "Monday, January 15, 2024"
formatDate(date, 'de-DE', { dateStyle: 'full' });
// "Montag, 15. Januar 2024"
```

### Relative Time

```tsx
const rtf = new Intl.RelativeTimeFormat(locale, { numeric: 'auto' });

rtf.format(-1, 'day');    // "yesterday" (en), "gestern" (de)
rtf.format(2, 'hour');    // "in 2 hours" (en), "in 2 Stunden" (de)
rtf.format(-3, 'month');  // "3 months ago" (en), "vor 3 Monaten" (de)
```

### Time Zones

```tsx
// Always store in UTC, display in user's timezone
function formatWithTimezone(date: Date, locale: string, timezone: string) {
  return new Intl.DateTimeFormat(locale, {
    dateStyle: 'medium',
    timeStyle: 'short',
    timeZone: timezone,
  }).format(date);
}

// formatWithTimezone(date, 'en-US', 'America/New_York')
// → "Jan 15, 2024, 2:30 PM"

// formatWithTimezone(date, 'en-US', 'Europe/London')
// → "Jan 15, 2024, 7:30 PM"
```

## RTL (Right-to-Left) Support

### Enabling RTL

```tsx
// Detect RTL languages
const rtlLanguages = ['ar', 'he', 'fa', 'ur'];
const isRTL = rtlLanguages.includes(locale.split('-')[0]);

// Set document direction
document.documentElement.dir = isRTL ? 'rtl' : 'ltr';
document.documentElement.lang = locale;
```

### CSS for RTL

```css
/* Use logical properties instead of physical */
/* ✗ Bad: Physical properties */
.sidebar {
  margin-left: 16px;
  padding-right: 8px;
  text-align: left;
  border-left: 1px solid;
}

/* ✓ Good: Logical properties */
.sidebar {
  margin-inline-start: 16px;
  padding-inline-end: 8px;
  text-align: start;
  border-inline-start: 1px solid;
}
```

### Logical Property Reference

```
┌─────────────────────┬───────────────────────┬─────────────────────┐
│ Physical (avoid)    │ Logical (use)         │ RTL behavior        │
├─────────────────────┼───────────────────────┼─────────────────────┤
│ margin-left         │ margin-inline-start   │ Becomes right       │
│ margin-right        │ margin-inline-end     │ Becomes left        │
│ padding-left        │ padding-inline-start  │ Becomes right       │
│ padding-right       │ padding-inline-end    │ Becomes left        │
│ left                │ inset-inline-start    │ Becomes right       │
│ right               │ inset-inline-end      │ Becomes left        │
│ text-align: left    │ text-align: start     │ Becomes right       │
│ text-align: right   │ text-align: end       │ Becomes left        │
│ border-left         │ border-inline-start   │ Becomes right       │
│ float: left         │ float: inline-start   │ Becomes right       │
└─────────────────────┴───────────────────────┴─────────────────────┘
```

### RTL Layout Patterns

```
LTR Layout:
┌────────────────────────────────────────────────────┐
│ [Logo]  Home  Products  About          [Sign In]  │
├───────────┬────────────────────────────────────────┤
│           │                                        │
│ Sidebar   │           Main Content                 │
│   Nav 1   │                                        │
│   Nav 2   │                                        │
│           │                                        │
└───────────┴────────────────────────────────────────┘

RTL Layout (automatically with logical properties):
┌────────────────────────────────────────────────────┐
│  [تسجيل]          حول  المنتجات  الرئيسية  [شعار] │
├────────────────────────────────────────────┬───────┤
│                                            │       │
│           المحتوى الرئيسي                  │ الشريط│
│                                            │ الجانبي│
│                                            │       │
└────────────────────────────────────────────┴───────┘
```

### Bidirectional Text

```tsx
// Use CSS for mixed-direction content
<p dir="auto">
  {/* Browser determines direction from content */}
  {message}
</p>

// Isolate embedded text with different direction
<p>
  The product name is <bdi>{arabicProductName}</bdi> and costs $50.
</p>

// Force direction for specific elements
<span dir="ltr">{phoneNumber}</span>  // Phone numbers always LTR
<span dir="ltr">{email}</span>        // Emails always LTR
```

## Text Expansion

### Expansion Guidelines

```
English text typically expands when translated:

┌───────────────┬─────────────────────────────────┐
│ Source length │ Expected expansion              │
├───────────────┼─────────────────────────────────┤
│ 1-10 chars    │ 200-300% (3x)                   │
│ 11-20 chars   │ 180-200% (2x)                   │
│ 21-30 chars   │ 160-180%                        │
│ 31-50 chars   │ 140-160%                        │
│ 51-70 chars   │ 130-140%                        │
│ 70+ chars     │ 120-130%                        │
└───────────────┴─────────────────────────────────┘

Example:
"Save" (4 chars) → "Speichern" (9 chars, German) = 225%
"Settings" (8 chars) → "Einstellungen" (13 chars) = 163%
```

### Design for Expansion

```
✗ Bad: Fixed-width buttons
┌──────────┐
│   Save   │
└──────────┘

✓ Good: Flexible buttons with padding
┌────────────────────────┐
│      Speichern        │
└────────────────────────┘

✗ Bad: Truncated text in fixed containers
┌──────────────────┐
│ Einstellun...    │
└──────────────────┘

✓ Good: Responsive containers or wrapping
┌────────────────────────┐
│ Einstellungen          │
└────────────────────────┘
```

### CSS for Text Expansion

```css
/* Allow buttons to grow */
.button {
  min-width: fit-content;
  padding-inline: 16px;
  white-space: nowrap;
}

/* Allow text to wrap in constrained spaces */
.nav-item {
  max-width: 200px;
  overflow-wrap: break-word;
  hyphens: auto;
}

/* Test with pseudo-localization */
[data-pseudo-locale] {
  word-spacing: 0.5em;
  letter-spacing: 0.1em;
}
```

## React Implementation

### Setup with react-i18next

```tsx
// i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import Backend from 'i18next-http-backend';
import LanguageDetector from 'i18next-browser-languagedetector';

i18n
  .use(Backend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'en',
    supportedLngs: ['en', 'de', 'ja', 'ar'],
    ns: ['common', 'auth', 'dashboard'],
    defaultNS: 'common',
    interpolation: {
      escapeValue: false, // React already escapes
    },
    detection: {
      order: ['querystring', 'cookie', 'localStorage', 'navigator'],
      caches: ['cookie', 'localStorage'],
    },
  });

export default i18n;
```

### Translation Hook

```tsx
import { useTranslation } from 'react-i18next';

function WelcomeMessage({ user }) {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t('welcome.title')}</h1>
      <p>{t('welcome.greeting', { name: user.name })}</p>
      <p>{t('welcome.itemCount', { count: user.items.length })}</p>
    </div>
  );
}
```

### Language Switcher

```tsx
function LanguageSwitcher() {
  const { i18n } = useTranslation();

  const languages = [
    { code: 'en', name: 'English', nativeName: 'English' },
    { code: 'de', name: 'German', nativeName: 'Deutsch' },
    { code: 'ja', name: 'Japanese', nativeName: '日本語' },
    { code: 'ar', name: 'Arabic', nativeName: 'العربية' },
  ];

  const handleChange = async (langCode: string) => {
    await i18n.changeLanguage(langCode);
    document.documentElement.lang = langCode;
    document.documentElement.dir = ['ar', 'he'].includes(langCode) ? 'rtl' : 'ltr';
  };

  return (
    <select
      value={i18n.language}
      onChange={(e) => handleChange(e.target.value)}
      aria-label={t('common.selectLanguage')}
    >
      {languages.map((lang) => (
        <option key={lang.code} value={lang.code}>
          {lang.nativeName}
        </option>
      ))}
    </select>
  );
}
```

### Formatted Components

```tsx
// Date formatting component
function FormattedDate({ date, options }: { date: Date; options?: Intl.DateTimeFormatOptions }) {
  const { i18n } = useTranslation();
  const formatted = new Intl.DateTimeFormat(i18n.language, options).format(date);
  return <time dateTime={date.toISOString()}>{formatted}</time>;
}

// Currency formatting component
function FormattedCurrency({ amount, currency }: { amount: number; currency: string }) {
  const { i18n } = useTranslation();
  const formatted = new Intl.NumberFormat(i18n.language, {
    style: 'currency',
    currency,
  }).format(amount);
  return <span>{formatted}</span>;
}

// Relative time component
function RelativeTime({ date }: { date: Date }) {
  const { i18n } = useTranslation();
  const rtf = new Intl.RelativeTimeFormat(i18n.language, { numeric: 'auto' });

  const diff = date.getTime() - Date.now();
  const days = Math.round(diff / (1000 * 60 * 60 * 24));

  return <time dateTime={date.toISOString()}>{rtf.format(days, 'day')}</time>;
}
```

## Testing

### Pseudo-Localization

```tsx
// Transform text to test for i18n issues
function pseudoLocalize(text: string): string {
  const map: Record<string, string> = {
    a: 'à', e: 'é', i: 'ì', o: 'ó', u: 'ù',
    A: 'À', E: 'É', I: 'Ì', O: 'Ó', U: 'Ù',
  };

  const transformed = text
    .split('')
    .map(char => map[char] || char)
    .join('');

  // Add padding to simulate expansion
  const padding = '~'.repeat(Math.ceil(text.length * 0.3));

  return `[${transformed}${padding}]`;
}

// "Settings" → "[Séttìngs~~~]"
```

```
□ All user-visible strings extracted to translation files
□ No hardcoded strings in components
□ Dates/numbers use Intl formatters
□ RTL layout works correctly

```

## Best Practices

### Do
```
✓ Use ICU message format for complex strings
✓ Use logical CSS properties (inline-start, not left)
✓ Use Intl APIs for formatting (not moment.js)
✓ Test with languages that expand (German) and contract (Chinese)
✓ Test RTL layout with real Arabic/Hebrew content
✓ Include context for translators in translation files
✓ Support locale variants (en-US, en-GB)
✓ Store user language preference
```

### Don't
```
✗ Concatenate translated strings
✗ Hardcode date/number formats
✗ Assume all languages read left-to-right
✗ Assume text length stays the same
✗ Use flags for language selection (use language names)
✗ Auto-detect language without user override
✗ Translate programmatically generated text
✗ Forget to translate alt text and ARIA labels
```
